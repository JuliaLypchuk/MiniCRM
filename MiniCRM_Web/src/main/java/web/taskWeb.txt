ТЗ для веб проекта (Часть 1).
“Форма авторизации + Форма регистрации”
Разработать веб-форму авторизации клиента на сайте.
В форме должно присутствовать поле для ввода логина и
пароля
Кнопка “Войти”, для осуществления регистрации.
Предусмотреть корректность введенных в поля значений.
После прохождения авторизации пользователь попадает в
кабинет клиента.
Если логин и пароль не верные, пользователь остается на
странице формы авторизации, и ему выводится сообщение
“Некорректный логин или пароль.”
Разработать веб-форму регистрации нового клиента на
сайте.
Предусмотреть в карточке клиента наличие таких полей:
Имя и фамилия, Номер телефона, email и домашний адрес.
Предусмотреть сохранение данных клиента в базу данных.
При сохранении проверять корректность заполнения полей в
форме.

ТЗ для веб проекта (Часть 2).
“Кабинет клиента + Форма подачи заявки на покупку”
Разработать страницу кабинета клиента.
Предусмотреть возможность внесения данных в карточку
клиента.
Предусмотреть возможность смены пароля, по алгоритму:
Вводим старый пароль, и два раза новый после чего
подтверждаем.
Предусмотреть возможность просмотра завершенных заказов.
Можно обозначить этот раздел как “История”.
*Предусмотреть интерфейс для мониторинга стадии
выполнения заказа.
Разработать страницу создания заявки.
Предусмотреть возможность выбора товара, в не
зависимости от того есть ли он в наличии.
Заявку формируем из нескольких позиций.
Сформированную заявку пользователь подтверждает, после
чего она попадает к менеджеру в CRM систему.
Контактные данные пользователя обязательно должны
передаваться вместе с заявкой.
Интерфейс менеджера (изменения):
Внести в интерфейс менеджера, возможность обрабатывать
заявки и формировать на их основании заказы.
Добавить pool-заявок куда они будут попадать для
обработки.
